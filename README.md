# MCP Server — ядро оркестратора ИИ-агентов

Учебный проект: ядро MCP-сервера на Python, которое умеет запускать
ИИ-агентов как подпроцессы, хранить их общую память и работать с
локальной LLM через [Ollama](https://ollama.com) по REST API.

## Возможности

- 🚀 Запуск ИИ-агентов как отдельных подпроцессов (обмен данными через
  stdin/stdout по протоколу "одна JSON-строка = одно сообщение")
- 🧠 Общая память агентов с автосохранением на диск (`workspace/memory.json`)
- 💬 Приём ввода от пользователя и передача его первому агенту в цепочке
- 🤖 Интеграция с Ollama через REST API (`/api/generate`, `/api/tags`)
- 📝 Логирование работы ядра (консоль + файлы в `logs/`)

## Структура проекта

```
.
├── core/                    # ядро оркестратора
│   ├── __init__.py

└── requirements.txt
```

## Требования

- Python 3.10+
- [Ollama](https://ollama.com) — установлена и запущена локально
- Хотя бы одна модель, скачанная в Ollama (например `llama3`)

## Установка

```bash
git clone <ссылка-на-репозиторий>
cd <папка-проекта>
pip install -r requirements.txt
```

Установи и запусти Ollama:

```bash
ollama serve
ollama pull llama3
```

## Запуск проверки

```bash
python run_test.py
```

Скрипт:
1. Запустит тестового агента (`agents/test_agent.py`) как подпроцесс;
2. Отправит ему три тестовых сообщения от "пользователя";
3. Выведет содержимое общей памяти агентов;
4. Сделает тестовый запрос к Ollama и покажет список доступных моделей и ответ модели.

## Протокол общения ядра с агентом

Ядро → агент (stdin):
```json
{"type": "user_input", "text": "...", "memory_snapshot": [...]}
```

Агент → ядро (stdout):
```json
{"type": "agent_response", "agent": "<имя агента>", "text": "..."}
```

Подробные правила — в [`workspace/rules/rules.md`](workspace/rules/rules.md).

## Основные методы `Orchestrator`

| Метод | Назначение |
|---|---|
| `start_agent(name, script_path)` | Запустить агента как подпроцесс |
| `stop_agent(name)` / `stop_all_agents()` | Остановить агента(ов) |
| `handle_user_input(text)` | Передать ввод пользователя первому агенту в цепочке |
| `remember(key, value)` / `recall(key)` | Чтение/запись общей памяти агентов |
| `query_ollama(prompt, model=...)` | Запрос к локальной LLM через Ollama |
| `list_ollama_models()` | Список установленных в Ollama моделей |
